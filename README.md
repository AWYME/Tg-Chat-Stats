# 📊 TG Chat Stats — разбор Telegram-логов как у профи

**Ты когда-нибудь хотел узнать, кто в чате главный болтун, кто чаще всех ставит эмодзи, а кто пишет романы по 1000 символов?**  
Этот скрипт читает JSON-экспорт Telegram Desktop и выдаёт **полный маркдаун-отчёт** по каждому участнику. Интонации, топ слов,
самые длинные сообщения, часы активности и даже лексическое разнообразие — всё внутри.

## 🚀 Возможности

### По чату в целом
- Общее количество сообщений
- Топ-20 слов (с фильтрацией стоп-слов)
- Активность по часам (UTC) + диаграмма
- Медиа-статистика (фото, видео, файлы и т.д.)
- Интонационный профиль чата (вопросы, восклицания, нейтрал, многоточия)

### По каждому участнику (от 10 сообщений)
- Количество сообщений, средняя длина, процент эмодзи
- Лексическое разнообразие (% уникальных слов)
- Персональная активность по часам
- Топ-10 самых частых слов
- Топ-10 **самых длинных сообщений** (с датой и обрезанным текстом)
- Интонационный портрет (в каком настроении писал)
- Часы пиковой активности

### Допнительно
- Работает с любым экспортом Telegram (формат JSON)
- Не требует внешних библиотек — только стандартный Python
- Результат — красивый Markdown, который можно открыть в браузере, Typora или VS Code

## 🛠 Требования
- **Python 3.6** или выше
- Экспортированный JSON-файл чата из Telegram Desktop

## 📦 Установка

1. Скачай скрипт `chat_stats_advanced.py`
2. Положи его в любую папку
3. (Опционально) Убедись, что Python установлен:
   ```bash python --version```

## 📁 Как получить JSON из Telegram

1. Открой **Telegram Desktop**
2. Зайди в **Настройки** → **Продвинутые** → **Экспорт данных**
3. Выбери нужный чат (или все чаты — потом скрипт разберёт)
4. В форматах экспорта выбери **JSON** (машиночитаемый)
5. Нажми «Экспорт» и дождись завершения
6. В папке экспорта найдёшь `result.json` (или `messages.json`)

## ▶️ Запуск

Открой терминал (командную строку) и выполни:

```bash
python chat_stats_advanced.py путь/к/result.json
```

Можно указать имя выходного файла:

```bash
python chat_stats_advanced.py result.json my_chat_stats.md
```

Если файл лежит в той же папке, что и скрипт, достаточно:

```bash
python chat_stats_advanced.py result.json
```

Скрипт создаст рядом `result_advanced_stats.md` (или имя, которое ты указал).

## ⚙️ Настройки под себя

Внутри скрипта в самом низу есть параметр `min_messages_per_user`.  
По умолчанию = 10 — в отчёт попадают только те, кто написал ≥10 сообщений.

Чтобы изменить, открой `chat_stats_advanced.py` и в строке вызова функции замени число:

```python
analyze_telegram_json(input_file, output_file, min_messages_per_user=3)
```

Теперь в отчёте будут даже самые молчаливые.

## 📌 Важные замечания

- **Часовой пояс** — все часы выводятся в **UTC** (такими Telegram отдаёт даты). Если твой чат живёт в Москве или Нью-Йорке — прибавь/отними в уме.
- **Стоп-слова** — предустановлен набор для русского и английского. Можно добавить свои в список `STOP_WORDS`.
- **Длинные сообщения** — обрезаются до 200 символов для читаемости, но в расчётах участвуют полностью.
- **Эмодзи** — распознаются стандартные Unicode-последовательности (смайлы, флаги, символы). Не каждый иероглиф посчитается за эмодзи, но 99% попадают.
- **Интонация** — определяется по последнему знаку препинания в сообщении. Многоточие, вопросы, восклицания, точка. Остальное — «нейтральная».

## 🧪 Проблемы и их решение

| Ошибка | Что делать |
|--------|-------------|
| `FileNotFoundError` | Проверь путь к JSON. Используй полный путь или положи файл рядом со скриптом. |
| `KeyError: 'messages'` | Возможно, у тебя экспорт старого формата. Скрипт всё равно работает — он умеет читать и список сообщений напрямую. |
| В маркдауне обрываются таблицы | Файл слишком большой для твоего просмотрщика (например, Typora обрезает >2MB). Открой в браузере или обычном блокноте. Или разбей файл (скриптом можно дополнить). |
| Неправильно определяются слова (слипаются) | Убедись, что в тексте нет левых символов. Скрипт чистит пунктуацию, но экзотику типа `вот_так` может не разделить. |
